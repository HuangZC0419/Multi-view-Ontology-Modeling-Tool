1. 快速入门

1.1 什么是本体建模

本体（Ontology）是知识工程中的核心概念，用于形式化地描述某一领域内的概念及其之间的关系。 简单来说，本体建模就是画出"什么东西"和"它们之间有什么关系"的一张图。

在科研、工业工程、企业知识管理等领域，本体建模可以帮助您：

  • 理清复杂系统中的概念层次（如设备与子设备、部件与零件）
  • 建立概念间的有向关系（如"控制""依赖""输入""输出"）
  • 定义关系规则，自动推导隐含关系（如传递规则）
  • 从文档、表格、图片中批量提取实体，快速构建知识图谱
假设您研究一个供热系统，您可以将"锅炉""换热器""水泵""阀门"作为本体， 用"输入""输出""控制"等关系将它们连接起来，形成一个清晰的知识网络。

1.3 第一个示例：三步创建本体图

让我们通过一个最简单的例子，快速上手整个系统：

  2. 创建第二个本体：再次点击 新增独立本体，输入名称"换热器"，确定。
  3. 建立关系：点击"锅炉"本体，再点击工具栏的 开启连线模式，然后在画布上点击"换热器"本体，在弹出的输入框中输入关系名"输出"，确定。一条带箭头的连线出现了。
恭喜！您已经创建了第一个本体关系图：锅炉 ──输出──▶ 换热器。现在您可以保存画布了。
连线模式开启后，您可以连续连线：每次点击目标本体并输入关系名后，系统会自动等待您选择下一个起始本体。按 Esc 键或点击"取消连线"按钮退出。

1.5 本体建模的基本概念速览

节点（Node / Ontology Entity）

节点代表一个概念或实体。在本系统中，每个节点有一个名称（如"锅炉"）、一个可选父节点、以及可选的键值对属性列表。节点在画布上显示为圆角卡片。

关系（Edge / Relation）

关系是节点之间的有向连接。每条关系有一个名称（如"控制"）、一个源节点和一个目标节点。关系分为两种：

  • 普通关系（relation）：由用户手动创建，可附加特征（对称、传递、函数）。
  • 父子关系（parent-child）：由系统自动管理，表示概念层次结构。

逻辑规则（Logic Rules）

逻辑规则是对关系的约束条件和推导规则的定义。分为互斥约束和顺承推理两类。规则在创建关系时自动检查执行。

项目（Project）

项目是图数据的隔离容器。不同项目拥有完全独立的本体图数据，互不影响。您可以同时管理多个项目，随时切换。

2. 本体操作

2.0 本体卡片结构详解

在开始操作之前，先了解画布上每个本体卡片的视觉结构：

卡片部位外观功能
顶部指示条 浅色细线，有子本体时为紫色 快速判断该节点是否有子本体。
图标区 蓝色菱形图标 统一的本体视觉标识。
名称显示 加粗字号，支持物理量符号格式化 本体的语义名称。如 Qp 等符号会自动渲染为斜体+下标。
ID 短码 灰色小字，显示前6位 UUID 用于开发调试或数据导出时定位节点。
折叠按钮 + 或 - 号，仅在有子节点时显示 切换子节点分支的显示/隐藏状态。
物理量符号格式化是自动进行的。系统目前支持以下格式：下划线模式（如 Q_p）、数字下标模式（如 CO2）、常见工程后缀模式（如 TDS_in 渲染为 TDSin）。 普通单词或中文名称不会受影响。

2.1 创建独立本体

独立本体是没有父本体的顶层概念节点。创建方式有三种：

方式操作
工具栏按钮 点击左侧工具栏 新增独立本体，输入名称后确认。本体会出现在画布中央偏上的随机位置。
右键菜单 在画布空白处右键，选择 在此新增独立本体。本体会精确地出现在您点击的位置。
OCR 导入 通过 OCR 识别结果批量导入实体（详见第5章）。
如果您希望在特定位置创建本体，请使用右键菜单方式。这对于规划大型本体图的结构非常有用。

2.2 创建子本体

子本体表示概念上的从属关系（如"水泵"是"供热系统"的子本体）。子本体与父本体之间会自动建立一条绿色父子连线。

  1. 选中父本体：点击画布上的一个本体卡片，使其高亮显示。
  2. 右键添加子本体：在该本体上右键，选择 新增子本体。
  3. 输入名称：在弹出的对话框中输入子本体的名称（如"循环泵"），确定。

子本体会自动放置在父本体的正下方，并在两者之间显示一条绿色的正交折线，标明"父子关系"。

场景：您有一个"供热系统"本体，想描述其子系统。
右键"供热系统"，依次添加子本体"热源子系统""输配子系统""末端子系统"，形成三层树状概念结构。

2.3 将已有本体转为子本体

如果您已经创建了两个独立本体，后来发现它们应该是父子关系，可以使用"子本体模式"来重新挂载：

  1. 开启子本体模式：选中待移动的本体，点击工具栏的 转为子本体模式（或在该本体上右键选择 将此节点转为子本体）。
  2. 选择新父本体：在画布上点击要作为其新父本体的节点。系统会自动完成挂载。
系统会自动检查并防止循环引用：如果您尝试将节点挂载到它自己的后代节点下，操作会被拒绝。

2.4 移动本体

直接拖拽画布上的本体卡片即可移动其位置。松开鼠标后，位置会自动保存到后端。连线会实时跟随节点移动。

拖拽时，所有相关连线会实时更新位置。请注意，父子关系的连线使用正交折线（直角线段），普通关系连线使用平滑曲线。

2.5 编辑本体名称

点击本体卡片，右侧会出现详情面板。在"名称"输入框中修改文本，修改即自动保存。

2.6 编辑本体属性

每个本体都可以附加键值对属性（类似知识图谱中的 Data Properties）。在详情面板中：

  1. 点击属性区域的 ＋ 按钮添加一行新属性。
  2. 在"键"列输入属性名（如"功率"），在"值"列输入属性值（如"500kW"）。
  3. 修改后自动保存。点击右侧 × 按钮可删除该属性。
场景：您有一个"水泵"本体，想记录它的技术参数。
添加属性：功率 → 500kW、扬程 → 30m、转速 → 2900rpm。

2.7 折叠与展开子节点

当您的本体树变得庞大时，可以折叠某些分支以保持画布整洁：

  1. 在画布上为一个有子本体的节点卡片上右键。
  2. 选择 折叠子节点关闭该分支，或其相反操作 展开子节点。

被折叠的节点及其所有后代节点和连线会从画布上隐藏，但数据仍然保留，不会丢失。

当一个节点有子本体时，其卡片顶部会显示一条紫色指示条和一个 -/+ 按钮，方便您快速判断和切换折叠状态。

2.8 删除本体

  1. 点击画布上的本体卡片，右侧出现详情面板。
  2. 点击面板底部的 删除此本体 按钮。
  3. 在弹出的确认对话框中点击"确定清空"。
删除本体会同时删除所有与该节点相关的连线（包括作为源和目标的连线、父子关系连线）。此操作不可撤销，请谨慎操作。

3. 关系操作

3.0 关系的视觉表示详解

在深入操作之前，先了解画布上关系的视觉语言：

关系类型线条样式颜色箭头
普通有向关系 平滑贝塞尔曲线 靛蓝色 (#6366F1) 蓝色三角单向箭头
父子关系 正交折线（直角） 翠绿色 (#10B981) 绿色三角单向箭头
对称关系 平滑曲线 紫色 (#8B5CF6) 双向箭头（两端均有）
已选中关系 （保持原样式） 亮蓝色高亮 高亮箭头

每条关系连线上方有一个半透明白色圆角标签，显示关系名称。标签区域同时也是点击热区，方便选中关系进行编辑。

3.1 什么是关系

关系（Relation）是本体之间的有向连接，描述两个概念之间的语义联系。 在画布上，关系以带箭头的曲线显示，箭头的方向表示关系的方向（从源指向目标）。

3.2 创建有向关系

  1. 选中起始本体：在画布上点击您希望作为关系起点的本体卡片。
  2. 开启连线模式：点击工具栏的 开启连线模式 按钮（或在起始本体上右键选择 从此节点连线）。状态栏会提示"连线模式：请点击目标本体"。
  3. 点击目标本体：在画布上点击关系的终点本体。
  4. 命名关系：在弹出的对话框中输入关系名称（如"控制""依赖""输入"），确定。

一条彩色的曲线箭头会立即出现在两个本体之间。

场景：在工业流程中描述设备间的关系。
创建：控制器 ──控制──▶ 阀门、水泵 ──输入──▶ 换热器、 锅炉 ──输出──▶ 蒸汽管道。
连线模式支持连续操作：每次完成一条连线后，系统不会退出连线模式，您可以继续点击下一个起始本体进行连线。 按 Esc 或点击"取消连线"退出。

3.3 父子关系

父子关系是系统自动管理的特殊关系类型：

  • 当您创建子本体或通过"子本体模式"重新挂载节点时，系统会自动创建父子关系连线。
  • 父子关系在画布上以绿色正交折线显示，区别于普通关系的彩色曲线。
  • 父子关系不可手动创建或删除，它完全由节点的挂载结构决定。
  • 关系名固定为"父子关系"，不可修改。

3.4 关系特征

在右侧详情面板中，您可以为普通关系（非父子关系）勾选以下三种特征：

特征英文名含义视觉变化
对称 Symmetric 如果 A 与 B 存在此关系，则 B 与 A 也存在此关系（方向无关）。 连线两端都显示箭头（双箭头），颜色变为紫色。
传递 Transitive 如果 A→B 和 B→C 存在此关系，则 A→C 也隐含此关系。 （当前版本通过顺承推理规则自动推导，见第4章）
函数 Functional 对于每个源节点，通过此关系最多只能连接到一个目标节点。 （该约束在当前版本中用于语义标记）
场景：在社交网络中，朋友 关系是对称的（A 是 B 的朋友则 B 也是 A 的朋友）。
将其勾选为 Symmetric，连线上会显示双向箭头，直观表达这一语义。
要编辑关系特征，请点击画布上的连线（连线的文本标签区域最容易点击到），右侧详情面板会切换为"关系详情"，其中包含"特征"复选框。

3.5 编辑关系名称

点击连线（建议点击连线上方的文本标签区域），右侧详情面板会出现 关系名称 输入框，修改后自动保存。

3.6 删除关系

  1. 点击画布上的连线（文本标签区域）。
  2. 右侧详情面板底部点击 删除此关系。
  3. 确认后删除。
如果连线较细难以点击，可以尝试点击连线上方的关系名称文字标签（白色圆角背景上显示），该区域有更大的点击热区。

3.7 关系关系名称自动补全

在创建关系或编辑关系名称时，输入框支持自动补全功能。当您开始输入时，会显示当前画布中已存在的所有关系名称，方便您保持术语统一。

3.8 互斥约束检查（创建时自动触发）

如果您已设置了互斥约束规则（见第4章），当您尝试在同一组节点之间创建冲突关系时，系统会自动拦截并提示。例如：

如果您定义了"管理"与"控制"互斥，而 A→B 已存在"管理"关系，则再次创建 A→B 的"控制"关系时，系统会提示：
"互斥约束冲突：无法在已有 '管理' 关系上建立 '控制'"

3.9 连线的几何算法说明

了解系统如何绘制连线，有助于您理解一些视觉现象：

端点计算原理

每条连线的起止端点并非直接指向节点中心，而是通过射线与矩形边界求交算法精确落在节点卡片的边框上。算法以对方节点的中心为参考点，计算从本节点中心出发的射线与卡片矩形的交点，确保连线无论节点如何移动都不会穿透卡片。

曲线样式对比

  • 普通关系：使用二次贝塞尔曲线（Quadratic Bezier），控制点在两端点中点的正下方，偏移量随距离增大而增大，形成自然的上拱弧线。
  • 父子关系：使用正交折线（Orthogonal Line），先垂直走到中点，再水平走向终点，体现树状结构的清晰层级关系。
  • 对称关系：渲染为双向箭头（起止端都有箭头标记），视觉上表达"A与B互为关系"的语义。

点击热区设计

每条关系连线在可见线条之外，还有一条20px 宽的透明路径完全覆盖在线条上方，用于扩大可点击区域。如果您觉得直接点击线条困难，请尝试点击关系名称标签（白色圆角背景上的文字）。

当两个节点水平距离较近时（如小于 150px），曲线拱高会很小，关系名称标签可能与节点卡片重叠。此时建议将节点拖开一些，或放大画布（鼠标滚轮上滚）以找到标签位置进行点击。

3.10 多关系并行的处理

当两个节点之间存在多条关系（例如 A 同时"控制""依赖""输入"B），这些关系会在画布上显示为多条曲线。由于所有都是贝塞尔曲线，它们可能在视觉上叠加。

当前处理方式：

  • 所有关系独立绘制，不做路径偏移处理。
  • 每条关系的名称标签位于曲线的不同位置（通过不同比例插值计算），可以一定程度区分开来。
  • 通过点击不同标签来选中不同的关系进行编辑或删除。
  • 使用互斥约束规则可以有效减少不必要的冗余关系（见第4章）。

3.11 关系操作的最佳实践

  1. 统一命名：在整个项目中保持关系名称一致（如统一使用"控制"而非混用"控制""操控""管理"）。输入框的自动补全功能可以帮助您。
  2. 先定规则后连线：在开始大规模连线之前，先规划并设置好互斥约束和顺承推理规则，这样可以减少返工。
  3. 避免过度连接：只创建有实际语义的关系，避免为了"美观"连接不相关的节点。过多的关系会让画布混乱。
  4. 区分父子与普通关系：概念上的从属关系（整体-部分）应使用父子关系（通过子本体模式），而非普通关系。这样可以充分利用折叠功能和树状布局。
  5. 关系方向有意义：请确保箭头的方向符合语义（从主动者指向被动者、从整体指向部分、从因指向果）。

4. 关系逻辑规则

4.1 什么是关系逻辑规则

关系逻辑规则是本系统的高级特性，允许您定义关系之间的约束条件和推导逻辑， 从而在构建本体图时自动维护语义一致性。规则分为两类：

  • 互斥约束：规定两种关系不能同时存在于同一组节点之间。
  • 顺承推理：规定当两条关系"接续"时，可以自动推导出第三条关系（传递链）。

点击工具栏的 关系逻辑规则设置 按钮打开规则管理面板。

4.2 互斥约束

原理说明

互斥约束用于定义"关系 A 和关系 B 不能同时存在于同一有向对上"。 当您创建新关系时，系统会检查是否与已有关系冲突，如有冲突则阻止创建。

操作步骤

  1. 打开"关系逻辑规则设置"面板（点击工具栏 ⚙️ 关系逻辑规则设置）。
  2. 确保当前激活 互斥约束 标签页。
  3. 在"关系 A"输入框中输入第一个关系名（如"管理"），在"关系 B"输入框中输入第二个关系名（如"控制"）。
  4. 点击 添加规则。
场景：在一个组织架构本体中，一个部门只能被一个上级"管理"，"管理"和"控制"两种关系不应并存。
设置互斥规则：管理 ↔ 控制（互斥）。
之后，如果 A→B 已有"管理"关系，再尝试添加 A→B 的"控制"关系时，系统会自动拦截并提示冲突。
场景：在一个设备关系本体中，设备 A 对设备 B 的关系要么是"主用"，要么是"备用"，不能同时存在。
设置互斥规则：主用 ↔ 备用（互斥）。
互斥约束只检查同一方向（相同的 source 到相同的 target）。不同方向的关系不受影响。

删除互斥规则

在规则列表中找到要删除的规则，点击右侧的删除图标即可。

4.3 顺承推理

原理说明

顺承推理定义了关系传递规则：当 A→B 存在"关系 1"，且 B→C 存在"关系 2"时， 系统会自动推导出 A→C 存在"推导结果关系"。推导出的新关系是实际创建在画布上的，不是虚拟的。

操作步骤

  1. 打开"关系逻辑规则设置"面板。
  2. 切换到 顺承推理 标签页。
  3. 依次填写三个输入框：
    • 关系 1：第一段关系的名称（如"控制"）
    • 关系 2：第二段关系的名称（如"依赖"）
    • 推导结果：自动生成的关系名称（如"支配"）
  4. 点击 添加规则。

规则添加后，系统会立即扫描已有连线，对满足条件的组合自动推导出新关系。之后每次创建新关系时，也会实时触发推导。

场景：在一个工业控制本体中：
传感器 → 控制 → 控制器 → 依赖 → 执行器
设置顺承推理规则：控制 + 依赖 → 支配
系统会自动创建：传感器 ──支配──▶ 执行器。
场景：在供应链本体中：
供应商 → 供货 → 仓库 → 分发 → 零售点
设置规则：供货 + 分发 → 供应链路
系统会自动创建：供应商 ──供应链路──▶ 零售点。
顺承推理是实际的连线创建，不是虚拟标注。推导出的新关系会显示在画布上，也可被后续规则再次用于推导（传递闭包）。如果需要撤销，请手动删除推导出的关系。

删除推理规则

删除推理规则只会阻止未来自动推导，不会删除已经推导出的关系。如需清理已推导的关系，请手动找到并删除。

4.4 规则管理技巧

  • 规则面板顶部显示当前活跃的标签页和规则数量（数字徽章）。
  • 所有输入框支持自动补全（基于画布中已有的关系名），方便保持术语一致。
  • 建议先规划好规则再大规模连线，避免后期大量修改。
  • 互斥约束和顺承推理可以同时使用，彼此独立工作。
  • 如果规则不生效，请检查关系名称是否完全匹配（区分大小写和中英文）。

4.5 规则应用的高级场景

场景一：多层传递链

顺承推理规则支持传递闭包：如果规则"控制 + 依赖 → 支配"触发生成了新关系 A→C（支配），而 C 又通过另一条"控制"关系连接到 D，规则会继续触发，生成 A→D（支配），形成完整的传递链。

画布现有：传感器 ──控制──▶ 控制器 ──依赖──▶ 执行器 ──控制──▶ 被控对象
规则：控制 + 依赖 → 支配
推导结果：
第1步：传感器 ──支配──▶ 执行器
第2步（继续推导）：传感器 ──支配──▶ 被控对象（因为传感器→执行器是"支配"即"控制"的传递）

注意：当前版本仅在"控制"和"依赖"这两类原关系上触发，"支配"作为推导结果不会自动作为新一轮的触发条件。如果您需要更深层的传递，请创建额外的推理规则。

场景二：组合规则防止数据矛盾

同时使用互斥约束和顺承推理，可以构建自洽的本体网络。例如，在供应链本体中：

  1. 设置互斥规则：主供 ↔ 备供（一个供应商对下游不能同时是主供和备供）。
  2. 设置顺承规则：主供 + 依赖 → 关键路径。
  3. 当您添加"供应商A 主供 仓库X"后，系统会自动推导出关键路径；但如果您再尝试添加"供应商A 备供 仓库X"，互斥规则会拦截。

场景三：动态规则调整

规则可以在任何时间添加或删除。添加新规则后，系统会立即扫描所有已有连线，对满足条件的组合自动推导。删除规则只会阻止未来推导，不删除已推导的关系。

建议在项目初期规划好"关系名称字典"：列出您将在整个本体图中使用的所有关系名，然后基于此字典设计互斥约束和顺承推理规则。这样可以避免后期因为关系名称不一致导致规则失效。

4.6 规则面板布局指南

规则设置面板从右侧滑出，覆盖在画布上方。其布局分为三个区域：

区域内容
顶部 Header齿轮图标 + 标题"关系逻辑规则" + 关闭按钮（×）。关闭面板时不会丢失已保存的规则。
Tab 切换栏"互斥约束"和"顺承推理"两个标签页，右侧带数字徽章显示规则数量。
内容区表单添加区（上方）+ 规则列表（下方）。规则列表为空时显示引导提示。

5. OCR 导入

5.1 功能介绍

OCR 导入功能允许您从图片（PNG、JPG、BMP、WebP）或文本文件（TXT、CSV）中 自动提取实体，并批量导入画布作为本体节点。这大大加速了从纸质文档、扫描件、表格等资料构建本体图的过程。

当前支持两种识别引擎：

引擎状态说明
本地识别 ✅ 可用 使用 RapidOCR 引擎在本地运行，无需联网，适合常规文档、表格、文本图片的识别。
大模型识别 ⏸ 暂不可用 基于云端 LLM（大语言模型）的 OCR 能力，部署至内网环境后该功能已暂停。

5.2 本地识别完整流程

  1. 选择文件：在工具栏的"OCR 导入"区域，点击文件选择按钮，选择一张图片或一个 CSV/TXT 文件。您也可以点击 使用示例文件 test.png 快速体验。
  2. 开始识别：点击 本地识别 按钮。系统会显示"识别中..."状态，完成后会提示识别到的实体数量。
  3. 查看结果：点击 查看识别结果 按钮，打开结果弹窗。左侧显示原始文件预览，右侧显示提取到的实体列表。
  4. 选择实体：在右侧实体列表中，勾选您想要导入画布的实体。可以使用 全选 / 全不选 快速操作。
  5. 一键导入：点击 一键导入勾选项，系统会将选中的实体作为本体节点添加到画布上。
导入的实体将自动以网格布局排列在画布左上角（4列排列），避免全部叠加在一起。您随后可以自由拖拽调整位置。

5.3 使用示例文件

系统内置了一张示例图片 test.png，位于项目根目录。点击工具栏的 使用示例文件 test.png 按钮即可自动加载。加载成功后直接点击"本地识别"即可体验 OCR 导入功能，无需手动选择文件。

示例文件包含一些常见的物理量符号和工程术语，识别后您可以看到系统如何从图片中提取实体概念。

5.4 支持的识别模式

系统会根据识别内容自动判断模式：

模式触发条件提取策略
表格模式 检测到明显的行列结构（至少3行、3列） 将每个非空的、有意义的单元格作为独立实体提取。
句子模式 文本呈自然语言排列 使用结巴分词提取名词、专有名词、英文符号作为候选实体。
对于物理量符号（如 Q_p、CO2、TDS_in），系统会使用正则表达式优先提取，避免被分词工具错误切分。识别到的符号会自动格式化显示（如 Qp）。

5.5 支持的文件格式

格式后缀处理方式
图片.png, .jpg, .jpeg, .bmp, .webp使用 OCR 引擎识别图像中的文字。
文本文件.txt, .md直接读取文本内容，提取名词和实体。
CSV 表格.csv解析 CSV 表格，将每个单元格作为候选实体提取。

5.6 实体导入细节

  • 如果实体名称已在画布中存在，则会自动跳过，不会重复创建。
  • 导入完成后，状态栏会提示"已导入 X 项，跳过 Y 项"。
  • 默认导入位置为画布左上角，以 4 列网格方式排列。
  • 如果当前画布经过了平移或缩放，导入位置会相应调整。
CSV 文件编码请使用 UTF-8。如果使用其他编码，可能导致中文乱码。系统会尝试自动检测编码（UTF-8、GB18030、GBK、Big5 等），但不能保证 100% 正确。

5.7 OCR 识别模式深入解析

表格模式（Table Mode）

当系统检测到图片或 CSV 文件中有明显的行列结构时，会进入表格模式。判断标准：

  • 至少存在 3 行且 3 列。
  • 排列整齐的行占比至少 60%（排除少列的不规则行）。
  • 在表格模式下，每个有意义的单元格都会被提取为独立实体。纯数字、短标点符号会被自动过滤。
  • 超过 15 个字符且包含中文的单元格，会进一步通过结巴分词提取名词。
场景：您有一个设备清单表格，包含"设备名称""型号""功率""所属系统"四列。
系统会提取每个单元格的内容作为实体，例如"锅炉""WNS-200""500kW""热源子系统"各自成为独立本体。您可以随后选择需要的实体导入，或将所有实体导入后再手动删除不需要的节点。

句子模式（Sentence Mode）

当内容不满足表格条件时，系统进入句子模式：

  • 使用结巴分词（jieba + posseg）对文本进行词性标注。
  • 优先提取名词（n 开头）、动名词（vn）、英文词（eng）作为实体。
  • 对英文符号（如 Q_p、Am、TDS_in）使用正则表达式预提取，防止被分词工具拆散。
  • 如果提取结果过少，会回退到按逗号、句号、换行符分句的策略。

5.8 物理量符号的识别与处理

本系统对物理量符号（常见于工程、科研论文、实验报告）有专门的识别优化：

  • 正则匹配：字母开头 + 字母/数字/下划线 组合（如 Q_p、CO2、TDS_in、Am）。
  • 避免碎片化：结巴分词可能将 Q_p 拆为 Q、_、p 三个词元，系统会自动合并，不重复添加。
  • OCR 纠错：将常见的识别错误自动修正，例如 A_ 修正为 Am（OCR 可能将 m 末笔识别为下划线）。
  • 大模型模式下，还会去除 LaTeX 包裹（如 $Q_{p}$ 还原为 Q_p）。

5.9 CSV 文件处理特殊说明

CSV 文件有独特的处理逻辑：

  • 所有非空的、有意义的单元格都会被提取。
  • 纯数字单元格会被跳过（不会作为实体导入）。
  • 短标点符号（无字母、数字或中文）也会被跳过。
  • 超过 10 个字符的单元格内容，会用结巴分词进一步提取名词。
  • 形如"表5-1""图2-3"的标题格式会被自动过滤。

5.10 OCR 识别与本体导入的实际操练示例

综合示例：从一张设备清单图片构建本体图

源文件：一张包含以下内容的表格图片：

| 设备名称 | 型号 | 功率 | 所属系统 |
| 锅炉 | WNS-200 | 500kW | 热源子系统 |
| 换热器 | BH-150 | 300kW | 热源子系统 |
| 循环泵 | CP-100 | 75kW | 输配子系统 |

操作流程：
1. 选择该图片文件，点击"本地识别"
2. OCR 识别为表格模式，提取出 12 个实体候选
3. 在结果弹窗中，勾选您需要的实体（例如勾选所有设备名称）
4. 点击"一键导入"，4 个设备以网格布局出现在画布上
5. 拖拽调整位置，手动创建"输入""输出"等关系连线
6. 通过右键菜单为每个设备添加子本体，细化部件结构

6. 画布操作

6.1 平移画布

当本体图超出当前可视区域时，您可以通过以下方式平移视角：

方式操作
鼠标拖拽 在画布空白处按住鼠标左键并拖动。画布内容会跟随鼠标移动。
平移操作只在画布空白区域有效。如果在节点卡片上按下鼠标，会触发节点拖拽移动，而非画布平移。

6.2 缩放画布

支持 10% 到 500% 的缩放范围，以下三种方式均可缩放：

方式操作说明
鼠标滚轮 在画布上滚动滚轮 向上滚放大，向下滚缩小。以鼠标位置为中心缩放，体验流畅。
缩放按钮 点击画布右下角的 ＋ / － 按钮 每次以画布中心放大/缩小 20%。
重置按钮 点击画布右下角的 ⌂ 按钮 将缩放比例重置为 100%，同时将画布平移归零。

6.3 保存当前画布

点击工具栏的 💾 保存当前画布 按钮，将当前所有本体、关系和规则数据保存到后端项目文件中。以下操作也会自动触发保存：

  • 创建/删除本体或关系
  • 拖拽移动本体位置
  • 修改本体名称、属性
  • 修改关系名称、特征
  • 添加/删除逻辑规则
  • 清空画布
大部分编辑操作会自动同步保存到后端。手动点击"保存当前画布"按钮是一种显式的备份习惯，确保数据安全。

6.4 右键菜单

在画布上的不同位置点击右键，会弹出不同的上下文菜单：

右键位置菜单项功能
空白处 在此新增独立本体 在点击位置创建新本体。
本体卡片上 新增子本体 为该本体创建一个子节点。
从此节点连线 以该节点为起点，快速进入连线模式。
将此节点转为子本体 将该节点挂载到另一个节点下。
折叠/展开子节点 隐藏或显示该节点的所有后代节点（仅当有子节点时显示）。
在画布任意位置点击，即可关闭右键菜单。按 Esc 键同样可以关闭菜单。

6.5 一键清空画布

点击工具栏的 🗑️ 一键清空画布，会弹出确认对话框。确认后，当前项目的所有本体和关系数据将被全部删除。此操作不可撤销。

清空操作会删除当前项目的全部数据（包括节点、关系、规则）。但不会删除其他项目的数据。操作前请确认您已切换到正确项目。

6.6 刷新画布

如果画布状态与预期不一致（例如显示异常），可以点击工具栏底部的 ↻ 刷新画布 按钮，从后端重新加载当前项目的数据。

6.7 网格背景

画布背景采用点阵网格设计，网格会随缩放比例自动调整点间距，帮助您在视觉上判断节点的相对位置和间距。

6.8 画布操作的性能提示

  • 拖拽节点时，所有相关连线会实时更新位置。系统使用了一个拖拽计数器（dragTick）来强制每次鼠标移动时重新计算连线端点。在大量节点（100+）的场景下，如果出现卡顿，属于正常现象。
  • 缩放变换使用 CSS transform: scale() 实现，浏览器会利用 GPU 加速渲染，性能表现良好。
  • 被折叠隐藏的节点和连线不会在 SVG 中渲染，从而减轻大量节点时的渲染负担。
  • 如果您需要处理数百个节点，建议利用父子关系和折叠功能将节点组织成树状结构，按需展开分支。

6.9 画布状态的持久化

画布的平移偏移量（viewOffset）和缩放比例（zoom）不会保存到后端。每次刷新页面或切换项目时，画布会恢复到默认视图（缩放 100%，平移归零）。如需多次查看同一区域，您可以：

  • 记录缩放比例和大致平移位置，以便下次快速定位。
  • 将相关节点拖拽到画布默认可见区域（左上角附近），减少每次手动平移的需要。

6.10 缩放控件详解

画布右下角的缩放控件提供了三种操作，其各自的行为如下：

按钮功能缩放步长范围
＋ 放大以画布中心为基准放大 20%×1.2 每次最大 500%
⌂ 重置恢复缩放为 100%，同时将平移归零直接复位回到初始状态
－ 缩小以画布中心为基准缩小 20%÷1.2 每次最小 10%
使用鼠标滚轮缩放时，缩放以鼠标指针位置为中心进行，体验比按钮缩放更直观。建议日常使用滚轮，仅在需要精确复位时使用按钮。

7. 项目管理

7.1 项目概述

系统支持多项目管理，每个项目拥有独立的本体图数据（节点、关系和规则）。 项目数据存储在 backend/projects/ 目录下，每个项目一个 JSON 文件。

项目选择器位于侧边栏顶部标题下方，是一个带 ☰ 图标的横条，显示当前项目名称和下拉箭头。 点击即可展开下拉菜单，无需额外面板空间。

7.2 切换项目

  1. 点击侧边栏顶部的项目选择器横条。
  2. 下拉菜单列出所有项目，当前项目有蓝色圆点指示器。
  3. 点击目标项目名称，画布立即加载该项目数据。

7.3 新建项目

  1. 点击项目选择器打开下拉菜单。
  2. 点击底部的 + 新建项目 按钮。
  3. 在弹出的对话框中输入项目名称。
  4. 确定后，系统会自动创建项目并切换到新项目。
场景：您同时负责两个不同的研究课题。
创建项目"课题A-污水处理"和"课题B-能源管理"，分别存放不同的本体模型，互不干扰。

7.4 重命名项目

  1. 展开项目下拉菜单，将鼠标悬停在目标项目上。
  2. 右侧出现编辑图标（铅笔），点击进入行内编辑。
  3. 输入新名称，按 Enter 确认，或按 Esc 取消。

7.5 删除项目

  1. 展开项目下拉菜单，将鼠标悬停在目标项目上。
  2. 右侧出现删除图标（垃圾桶），点击弹出确认对话框。
  3. 确认后项目及其所有数据将被永久删除。
系统要求至少保留一个项目。如果只有一个项目，删除操作会被拒绝。删除项目后，会自动切换到列表中的第一个项目。

7.6 案例项目

系统预置了两个示例项目，帮助您快速上手：

项目名领域节点数边数

7.7 数据存储说明

项目数据存储在 backend/projects/ 目录下：

文件说明
projects/index.json 项目索引文件，记录所有项目的 ID、名称、创建时间和修改时间。
projects/{项目ID}.json 每个项目的完整图数据文件，包含 nodes、edges、inference_rules、mutex_rules。
您可以直接复制 projects/ 目录进行数据备份。这是最简单的备份方式（另见第8章"常见问题"中的备份说明）。

7.8 多项目协作建议

虽然本系统是单用户工具，但以下实践可以帮助您更有效地管理多个项目：

  • 命名规范：使用有意义的项目名称，如"课题-主题-版本"格式（例如"供热系统-管网模型-v2"）。
  • 定期备份：在重要操作前（如批量删除、大规模连线），先手动复制 projects/ 目录。
  • 项目迁移：如需将项目迁移到另一台机器，复制 projects/ 目录到新机器的对应路径即可，无需其他配置。
  • 版本管理：可以通过复制项目 JSON 文件并重命名来实现快照版本的保存。注意同步更新 index.json 中的记录。

7.9 项目时间戳说明

每个项目在创建时记录 created_at（创建时间），在每次保存图数据时更新 updated_at（最后修改时间）。这两个时间戳显示在项目列表中的项目名称下方，格式为中文日期（如"2026/5/14"）。

项目的时间戳仅在数据实际发生变化时才更新。仅切换项目或查看画布不会触发更新时间的变化。

7.10 初始项目与数据迁移

首次启动系统时，如果没有现成的项目数据，系统会自动创建一个名为"默认项目"的空白项目。此外，系统还会检查是否存在旧版数据文件（backend/data.json），如果有，会自动迁移到新的多项目存储结构中。

旧版数据迁移是一次性操作。迁移完成后，原始 data.json 文件会被自动删除。如果您的数据仍在该文件中，请确保后端有读写权限以完成迁移。

8. 常见问题

8.1 连线箭头被本体卡片遮挡

系统使用几何算法计算连线与矩形卡片的交点，确保连线端点精确落在卡片边框上，而非延伸到卡片内部。但以下情况可能导致视觉上的遮挡：

  • 节点间距过近：两个节点的卡片距离太近时，连线会非常短，箭头和标签可能叠加在一起。
  • 多个节点重叠：如果多个节点位置重叠或接近重叠，连线会被遮挡。

解决方案：将节点适当地拖拽分开，保持足够的间距。一般建议节点间水平距离至少 150 像素。

每条连线的上方有一个半透明白色圆角背景的关系名称标签。如果您看不到标签，可能是因为节点间距过小导致标签被截断。放大画布（滚轮上滚）可以看得更清楚。

8.2 点击连线没有反应（无法选中）

连线本身较细，可能难以精确点击。解决方式：

  • 请点击连线上方的关系名称文字标签（白色圆角背景），该区域有一个 20px 宽的无形点击热区。
  • 如果画面缩放比例太小导致连线太短，请使用滚轮放大画布后再操作。

8.3 顺承推理规则不生效

如果添加了推理规则但没有自动推导出新关系，请依次检查：

  1. 关系名称完全一致：规则中定义的关系名（如"控制"）必须与画布上实际存在的关系名称逐字逐符相同，包括大小写、空格、中英文标点。
  2. 路径方向正确：规则要求 A→B 存在"关系 1"，且 B→C 存在"关系 2"。请确认 B 节点是两个关系中的中间节点（前者的 target = 后者的 source）。
  3. 关系是 rule 中定义的顺序：规则中"关系 1"对应第一段（A→B），"关系 2"对应第二段（B→C），顺序不能颠倒。
  4. 不存在重复关系：如果推导出的关系已存在（相同 source、target、relation），系统不会重复创建。
  5. 节点未被折叠：如果中间节点或其两端关系被折叠隐藏，推理规则不会对隐藏节点生效（需要先展开）。
假设您定义了规则：控制 + 依赖 → 支配
如果画布上有：节点A ──控制──▶ 节点B 和 节点B ──依赖──▶ 节点C
系统会自动创建：节点A ──支配──▶ 节点C。
但如果关系名写错了（如"控制"写成了"掌控"），规则就不会生效。

8.4 互斥约束不生效

互斥约束仅在创建新关系时检查。不会对已有关系进行回溯检查。如果两条冲突关系在规则添加之前就已存在，系统不会自动删除其中一条。

解决方案：先添加互斥规则，再建立关系。或者手动删除其中一条冲突关系，再重新尝试创建。

互斥约束只检查相同节点对（same source → same target）上的关系。如果 A→B 有"管理"，而 B→A 有"控制"，不会被互斥规则拦截，因为方向不同。

8.5 OCR 识别结果为空或缺少内容

可能的原因及解决方法：

  • 图片质量差：模糊、倾斜、光照不均的图片对 OCR 引擎挑战较大。建议使用清晰、正面拍摄的图片。
  • 图片中文字太少：如果图片以图形为主、文字极少，识别结果会相应较少。
  • 表格列太少：表格模式需要至少3行3列才会被识别为表格。小表格会按句子模式处理。
  • 文件编码问题：CSV/TXT 文件请确保使用 UTF-8 编码。系统会尝试自动检测常见编码。

8.6 数据如何备份和恢复

备份方法

直接复制 backend/projects/ 整个目录即可完成备份。该目录包含：

  • index.json — 项目列表（名称、创建/修改时间）
  • {项目ID}.json — 每个项目的完整图数据

恢复方法

将备份的 projects/ 目录覆盖回去即可。恢复前请先关闭后端服务，覆盖后重启。

在项目之间复制数据

目前系统不支持跨项目复制粘贴。如需在两个项目间迁移数据，可以：

  1. 手动编辑 projects/ 目录下的 JSON 文件，将 nodes/edges 从源项目复制到目标项目文件中（注意保持 JSON 结构完整）。
  2. 或者通过 OCR 导入功能将同一份源数据重新导入到新项目中。
手动编辑 JSON 文件时请务必保持 JSON 结构完整，任何语法错误都会导致项目无法加载。建议先备份再编辑。

8.7 画布加载不出来或显示空白

请按以下顺序排查：

  1. 确认后端服务正在运行（默认 http://127.0.0.1:8000）。
  2. 检查状态栏的消息，确认没有网络错误提示。
  3. 尝试点击 ↻ 刷新画布 按钮。
  4. 检查项目列表是否正常加载。如果项目列表为空，请检查 backend/projects/ 目录是否存在且可读写。
  5. 如果使用了平移+缩放，画布可能被移到了视野之外。点击缩放控件中的 ⌂ 重置按钮恢复默认视图。

8.9 系统限制与注意事项

  • 本体和关系的数量理论上没有限制，但节点数量极大时（数千个以上），画布渲染性能可能下降。
  • 大模型 OCR 功能在部署至内网环境后已暂停使用，工具栏中的"大模型识别"按钮显示为禁用状态。
  • 系统不提供撤销/重做功能，删除操作请谨慎。建议定期手动备份 projects/ 目录。
  • 父子关系由系统自动管理，不能手动创建或删除父子关系连线。
  • 物理量符号的格式化目前支持特定的下标模式（如 Q_p、CO2、TDS_in），其他格式的符号不会自动美化。

8.10 大模型OCR状态查询

系统提供了一个 API 端点用于查询大模型 OCR 的可用状态：GET /api/ocr/llm-status。返回 JSON 格式：

{
  "enabled": false,
  "reason": "部署至内网环境，大模型识别功能已暂停使用"
}

当前部署在内网环境中，该功能已暂停。如需恢复，只需将后端代码中的全局变量 LLM_ENABLED 改为 True，并配置好 backend/LLM.env 中的 API 密钥即可。

8.11 状态栏消息解读

系统左下角的状态面板会实时显示当前操作的状态信息。消息类型如下：

包含"失败"字操作出错（显示红色）是，请检查网络或后端状态
"就绪"系统待命否，正常初始状态

8.12 从零开始构建一个完整的工程本体图

以下是端到端的完整操作流程，适用于初次使用者：

  1. 创建项目：点击"新建项目"，命名为"我的工程本体"。
  2. 先规划关系名：想好您需要哪些关系类型（如"控制""依赖""输入""输出""父子关系"）。
  3. 设置逻辑规则：打开规则面板，添加必要的互斥约束和顺承推理规则。
  4. 创建核心本体：通过工具栏按钮或右键菜单，创建主要的概念节点（顶层本体）。
  5. 创建子本体：右键各节点添加子本体，构建概念层次树。
  6. 建立关系连线：开启连线模式，为相关本体创建有向关系。
  7. 编辑属性：选中各节点，在右侧详情面板中添加键值对属性。
  8. 检查和调整：拖拽节点优化布局，利用折叠功能管理复杂分支。
  9. 保存并备份：点击保存按钮，并定期备份 projects/ 目录。

8.13 前端与后端的通信架构

了解系统架构有助于排查连接问题：

  • 后端：FastAPI 服务，默认运行在 http://0.0.0.0:8000，提供 RESTful API。
  • 前端：Vue3 + Vite 构建的单页应用，通过 fetch 调用后端 API。
  • API 地址：前端通过 VITE_API_BASE 环境变量或自动检测当前主机名来确定后端地址。
  • 数据格式：所有 API 通信使用 JSON 格式（文件上传使用 FormData）。
  • 项目数据：存储在 backend/projects/ 目录，每个项目一个 JSON 文件。

8.14 本体建模的理论背景简介

如果您对本体建模的理论背景感兴趣，以下基础概念可以帮助您更好地使用本系统：

什么是本体（Ontology）

在计算机科学中，本体是"对共享概念体系的形式化、明确的规范说明"。通俗理解：用一套统一的词汇和关系，清晰地描述某个领域内"有什么"和"它们之间有什么关系"。

本体建模的核心要素（Gruber 五要素）

要素英文在本系统中的应用
概念/类Classes/Concepts画布上的本体节点
关系Relations节点之间的有向连线
函数Functions表示为函数性（Functional）关系特征
公理Axioms体现为逻辑规则（互斥约束、顺承推理）
实例Instances画布上的具体本体节点（对应概念的具体化）

本体建模的应用场景

  • 工程系统建模：描述设备、管道、控制系统之间的物理和逻辑关系。
  • 知识图谱构建：构建结构化的领域知识库，支持问答和推理。
  • 供应链分析：描述供应商、产品、物流节点之间的依赖和约束关系。
  • 科研文献整理：从论文中提取核心概念和概念间的关系，形成知识网络。
  • 企业架构描述：描述组织、流程、信息系统之间的关联。

以下表格总结了本系统在不同场景下的推荐操作流程，供参考：

8.16 环境配置与启动说明

本系统由前端（Vue3 + Vite）和后端（Python FastAPI）两部分组成。完整的启动流程如下：

后端启动

cd backend
pip install -r requirements.txt
python app/main.py
# 服务默认运行在 http://0.0.0.0:8000

前端启动

cd frontend
npm install
npm run dev
# 开发服务器默认运行在 http://localhost:5173

API 地址配置

前端通过环境变量 VITE_API_BASE 确定后端 API 地址。如果前端和后端在同一台机器上运行，前端会自动检测并使用 http://{本机IP}:8000 作为 API 地址。如需自定义，请在 frontend/.env 文件中设置：

VITE_API_BASE=http://192.168.1.100:8000

部署到生产环境

前端构建后，dist/ 目录下的静态文件可以直接部署到 Nginx 或其他 Web 服务器：

cd frontend
npm run build
# 将 dist/ 目录部署到您的 Web 服务器

8.17 帮助文档访问方式

本帮助文档（help.html）部署在前端的 public/ 目录下。当您运行前端开发服务器后，可以通过以下 URL 访问：

http://localhost:5173/help.html

在生产部署中，该文件会随前端静态资源一并发布。您也可以在浏览器中直接打开该文件（无需服务器），但由于使用了内联样式和脚本，所有功能均可在本地文件协议下正常工作。
