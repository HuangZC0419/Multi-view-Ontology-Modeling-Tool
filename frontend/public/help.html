1. 快速入门

1.1 什么是本体建模系统

本系统是一套面向工程领域的本体建模与因果分析工具，帮助您将领域内的 概念（本体）及其之间的关系（连线）以可视化的方式组织起来， 形成清晰的知识网络。在此基础上，系统提供贝叶斯因果分析能力， 对概念间的因果链条进行概率推理。

核心应用场景包括：

  • 工业设备、工艺流程的概念层次建模
  • 从数据库和 Excel 表格自动导入实体构建知识图谱
  • 定义关系推理规则，自动推导隐含关系
  • 基于 Noisy-OR 模型的因果概率分析
  • 多视角浏览：管理者概览仪表盘、数据映射脑图、本体推理画布

1.2 登录系统

打开系统后，首先看到的是登录界面。输入用户名和密码后即可进入主界面。 系统支持两种角色：管理者（manager）和普通用户（user）。 管理者拥有全部操作权限（建模、导入、配置），普通用户为只读模式。 详见第2章 用户认证与权限。

1.4 三步创建第一张本体图

  1. 新建项目：点击左侧边栏顶部的项目选择器，点击底部的 "+ 新建项目"，输入项目名称并确认。
  2. 创建本体：切换到 建模 工作台标签，点击 新增节点，输入名称"锅炉"。 再重复此操作创建"换热器"。
  3. 建立关系：点击 新增关系，选择"画布点选"方式。先点击"锅炉"本体，再点击"换热器"本体， 在弹出的输入框中输入关系名"输出"，确定。一条带箭头的连线就出现了。
恭喜！您已创建第一个本体关系图：锅炉 ──输出──▶ 换热器。 点击 保存当前画布 按钮保存您的工作。

1.5 核心概念速览

节点（Node / Ontology Entity）

节点代表一个概念或实体。每个节点有一个名称、一个可选父节点以及可选的键值对属性列表。节点在画布上显示为圆角卡片。

关系（Edge / Relation）

关系是节点之间的有向连接。每条关系有一个名称、一个源节点和一个目标节点。关系分为两种：

  • 普通关系（relation）：用户手动创建，可附加特征（对称、传递、函数）和因果权重。
  • 父子关系（parent-child）：系统自动管理，表示概念从属层次。

逻辑规则（Logic Rules）

逻辑规则是对关系的约束条件和推导规则的定义，分为互斥约束和顺承推理两类。规则在创建关系时自动检查执行。

工作台（Workbench）

左侧边栏的工作台标签页将操作分为三个场景：数据导入（从外部数据源获取实体）、建模（创建/编辑本体与关系）、分析（贝叶斯因果分析）。

多视角（Perspective）

主内容区顶部有三个视角标签：概览（管理者仪表盘）、数据映射（数据血缘脑图）、本体推理（交互式建模画布）。每个视角提供不同的信息呈现方式。

项目（Project）

项目是图数据的隔离容器。不同项目拥有完全独立的本体图数据，互不影响。您可以同时管理多个项目，随时切换。

2. 用户认证与权限

2.1 登录系统

启动系统后，首先进入登录界面。输入用户名和密码，点击登录按钮即可进入主界面。 用户账户信息存储在后台的 users.xlsx 文件中，包含用户名、密码、角色和显示名称四个字段。

默认演示账号：
· 管理员 — 用户名 admin，密码 admin123，角色 manager
· 普通用户 — 用户名 user，密码 user123，角色 user
登录成功后，系统会将登录状态保存在浏览器的 localStorage 中。下次打开页面时，系统会自动验证会话有效性，如果有效则跳过登录界面直接进入主界面。

2.2 角色权限

系统支持两种角色，权限如下：

功能管理者 (manager)普通用户 (user)

2.3 登出与会话保持

点击左侧边栏底部的用户信息区域的退出按钮即可登出。登出后会话令牌失效，返回登录界面。

系统使用服务端会话令牌进行认证。每次登录生成一个新的 token，登出或 token 过期后需要重新登录。

3. 项目管理

3.1 项目概述

系统支持多项目管理，每个项目拥有独立的本体图数据（节点、关系、规则、权重配置）。 项目数据存储在 backend/projects/ 目录下，每个项目对应一个 JSON 文件。

项目选择器位于左侧边栏顶部标题下方，是一个带三横线图标（☰）的横条，显示当前项目名称和下拉箭头。 点击即可展开下拉菜单。

3.2 切换项目

  1. 点击侧边栏顶部的项目选择器横条。
  2. 下拉菜单列出所有项目，当前项目前方有蓝色圆点指示。
  3. 点击目标项目名称，画布立即加载该项目数据。

3.3 新建项目

  1. 点击项目选择器打开下拉菜单。
  2. 点击底部的 + 新建项目 按钮（仅管理者可见）。
  3. 在弹出的对话框中输入项目名称。
  4. 确定后，系统自动创建并切换到新项目。

3.4 重命名项目

  1. 展开项目下拉菜单，将鼠标悬停在目标项目上。
  2. 右侧出现编辑图标（铅笔），点击进入行内编辑。
  3. 输入新名称，按 Enter 确认。按 Esc 放弃。

3.5 删除项目

  1. 展开项目下拉菜单，将鼠标悬停在目标项目上。
  2. 右侧出现删除图标（垃圾桶），点击弹出确认对话框。
  3. 确认后项目及其所有数据将被永久删除。
系统要求至少保留一个项目。如果只有一个项目，删除操作会被拒绝。删除后自动切换到列表中的第一个项目。

3.6 演示项目

系统内置演示项目帮助快速上手：

项目名领域包含内容
机械加工工艺案例机械制造包含完整的加工工艺本体节点、关系链、逻辑规则与权重配置

4. 数据导入

4.1 功能概述

数据导入功能位于左侧边栏的"数据导入"工作台标签页，支持从数据库和文件两种数据源 自动提取表结构和列信息，批量生成本体节点。仅管理者可用。

4.2 数据库连接导入

系统支持两种数据库连接模式：

模式说明适用场景
模拟达梦基于 SQLite 的本地模拟数据库，无需任何连接参数。内置 5 张表、30 个列，模拟典型工程数据库结构演示、测试、无需真实数据库
真实达梦连接真实的达梦 DM8 数据库，需要主机、端口、用户名、密码等参数连接实际生产/测试数据库

操作步骤

  1. 在"数据导入"标签页中，点击 连接数据库 按钮。
  2. 选择连接模式（模拟或真实）。真实模式需填写连接参数并先测试连接。
  3. 连接成功后，系统自动加载数据库中的表列表。
  4. 点击目标表名，展开查看表中的列信息。
  5. 可选：点击预览查看表中的前几行数据。
模拟达梦模式无需任何前置条件，适合快速体验数据导入功能。数据库文件位于 backend/projects/sim_dameng.db。

4.3 文件上传导入

支持上传 Excel（.xlsx / .xls）和 CSV（.csv）文件，自动解析 Sheet/表格结构。

  1. 在"数据导入"标签页中，点击 上传文件 按钮。
  2. 将文件拖拽到上传区域，或点击选择文件。
  3. 上传成功后，系统自动解析文件结构（Excel 的 Sheet 名称、CSV 的列名）。
  4. 可选：点击预览查看文件中的前几行数据。
CSV 文件请使用 UTF-8 编码。系统会尝试自动检测常见编码（GB18030、GBK、Big5 等），但不能保证 100% 正确。

4.4 Schema 预览与导入

无论是数据库连接还是文件上传，解析完成后都会进入Schema 预览界面：

  1. 左侧显示树形结构：数据源 → 表/Sheet → 列，每个节点可展开/折叠。
  2. 勾选需要导入的表和列（支持全选/单选/逐项选择）。
  3. 可在导入前修改表名和列名（行内编辑）。
  4. 确认无误后，点击 导入勾选项。

导入后，每张表会作为独立的本体节点添加到画布上，列作为该节点的属性或子节点。

4.5 本体关系表模板导入

除了表结构导入，系统还支持"本体关系表模板"导入模式。这种方式直接按照实体—关系—属性的模板格式导入， 适用于已经按本体建模规范整理好的 Excel 文件。

模板要求 Excel 文件包含以下命名 Sheet：

Sheet 名称内容格式
实体关系定义本体之间关系的三元组实体A、关系名、实体B
实体属性定义每个实体的键值对属性实体名、属性键、属性值
推理规则定义顺承推理规则关系1、关系2、推导结果
互斥规则定义互斥约束规则关系A、关系B
例如，一个机械加工工艺的模板文件可以定义：
· 实体关系：车削 → 输入 → 工件毛坯、车削 → 使用 → 车床
· 实体属性：车床 → 类型 → 数控、车床 → 精度 → IT6
· 推理规则：输入 + 使用 → 工艺链路
· 互斥规则：主用 ↔ 备用

4.6 OCR 图片识别导入

除了结构化数据导入，系统还支持OCR 光学字符识别功能，从图片（PNG、JPG、BMP、WebP）或 文本文件（TXT、CSV）中自动提取实体，并批量导入画布作为本体节点。 这在从纸质文档扫描件、设备铭牌照片、表格截图等非结构化资料快速构建本体图时尤为有用。

识别流程

  1. 选择文件：在"数据导入"标签页的"OCR 图片识别"区域，点击文件选择按钮选取一张图片或文本文件。也可以点击 使用示例文件 test.png 快速体验。
  2. 开始识别：点击 本地识别 按钮，系统将调用 RapidOCR 引擎对文件内容进行文字识别和实体提取。识别过程中按钮显示"识别中..."。
  3. 查看结果：识别完成后点击 查看识别结果，弹出结果窗口——左侧显示原始文件预览（图片或文本），右侧列出提取到的实体候选列表。
  4. 选择并导入：在候选列表中勾选需要的实体（支持全选/全不选/逐项勾选），点击 一键导入勾选项。实体将以4 列网格布局排列在画布上。

两种识别模式

模式触发条件提取策略
表格模式检测到至少 3 行 × 3 列的规则行列结构将每个有意义的单元格作为独立实体提取。纯数字和短标点自动过滤。
句子模式内容呈自然语言排列，不满足表格条件使用结巴分词（jieba）提取名词、动名词、英文符号作为候选实体。对物理量符号（如 Q_p、CO₂）优先正则匹配，防止被拆分。

支持的文件格式

格式处理方式
.png / .jpg / .jpeg / .bmp / .webp使用 RapidOCR 引擎识别图像中的文字，自动判断表格/句子模式
.txt / .md直接读取文本内容，使用结巴分词提取名词和实体
.csv解析 CSV 表格，将每个单元格作为候选实体提取
如果实体名称在画布中已存在，则会自动跳过，避免重复创建。导入完成后状态栏提示"已导入 X 项，跳过 Y 项"。
场景：有一张设备清单表格的截图，包含"锅炉""换热器""循环泵""阀门"等设备名称。
1. 选择该截图文件 → 点击"本地识别"
2. OCR 识别为表格模式，提取出所有设备名称
3. 全选实体 → 一键导入 → 4 台设备以网格布局出现在画布上
4. 在建模标签中补充工艺关系连线，完成从图片到本体图的快速构建
CSV 文件请使用 UTF-8 编码。系统会尝试自动检测常见编码（UTF-8、GB18030、GBK、Big5、UTF-16），但不能保证 100% 正确。 大模型 OCR 识别功能在内网部署环境中暂不可用，工具栏中的"大模型识别"按钮为禁用状态。

5. 本体建模

5.0 本体卡片结构

在开始操作之前，先了解画布上每个本体卡片的视觉结构：

卡片部位外观功能
顶部指示条浅色细线，有子本体时为紫色快速判断该节点是否有子本体
图标区蓝色菱形图标统一的本体视觉标识
名称显示加粗字号本体的语义名称
ID 短码灰色小字，显示前 6 位 UUID用于调试或数据导出时定位节点
折叠按钮+ / - 号，有子节点时显示切换子节点分支的显示/隐藏状态

5.1 创建独立本体

独立本体是没有父本体的顶层概念节点。切换到 建模 工作台标签后：

方式操作
工具栏按钮点击 新增节点，输入名称后确认。节点出现在画布中心偏上位置。
右键菜单在画布空白处右键，选择 在此新增独立本体。节点精确出现在点击位置。
数据导入通过数据导入功能批量导入（详见第4章）。
如果希望在特定位置创建本体，请使用右键菜单方式。

5.2 创建子本体

子本体表示概念上的从属关系（如"循环泵"是"输配子系统"的子本体）。子本体与父本体之间会自动建立绿色父子连线。

  1. 选中父本体：点击画布上的一个本体卡片，使其高亮。
  2. 右键添加子本体：在该本体上右键，选择 新增子本体。
  3. 输入名称：在弹出的对话框中输入子本体名称，确定。

子本体会自动放置在父本体的下方，两者之间显示一条绿色正交折线。

如果父节点处于折叠状态，创建子节点后会自动展开，方便您看到新创建的子节点。

5.3 将已有本体转为子本体

如果已经创建了两个独立本体，后来发现它们应该是父子关系：

  1. 右键本体：在待移动的本体上右键，选择 将此节点转为子本体。
  2. 选择新父本体：在画布上点击要作为其新父本体的节点。
系统会自动检查并防止循环引用。如果您尝试将节点挂载到它自己的后代节点下，操作会被拒绝。

5.4 移动本体

直接拖拽画布上的本体卡片即可移动位置。松开鼠标后位置自动保存。所有相关连线会实时跟随节点移动。

5.5 编辑本体名称与属性

点击本体卡片，右侧出现详情面板。可以在此编辑：

  • 名称：修改后自动保存。
  • 属性（键值对）：点击 + 添加新属性，在键/值列输入内容。修改自动保存。点击 × 删除该属性。
为"水泵"添加技术参数：功率 → 500kW、扬程 → 30m、转速 → 2900rpm。

5.6 折叠与展开子节点

当本体树变得庞大时，可以折叠某些分支：

  1. 在画布上右键一个有子本体的节点卡片。
  2. 选择 折叠子节点（或 展开子节点）。

被折叠的节点及其所有后代节点和连线从画布上隐藏，但数据不会丢失。节点卡片顶部的紫色指示条和 + / - 按钮可以帮助判断折叠状态。

5.7 删除本体

  1. 点击画布上的本体卡片，右侧出现详情面板。
  2. 点击面板底部的 删除此本体 按钮。
  3. 在弹出的确认对话框中确认。
删除本体会同时删除所有相关连线（包括源/目标连线和父子连线）。此操作不可撤销。

6.6 连线几何说明

  • 端点计算：连线起止点通过射线与矩形边界求交算法，精确落在节点卡片边框上，不会穿透卡片。

  • 父子关系：使用正交折线，先垂直后水平，体现树状层级。
  • 点击热区：每条连线有 20px 宽的透明路径覆盖，便于点击选中。
  • 多关系并行：两个节点间可存在多条关系，各关系名称标签位于曲线不同位置以便区分。

7. 关系逻辑规则

7.1 概述

关系逻辑规则定义关系之间的约束条件和推导逻辑。 点击建模标签中的 关系逻辑规则设置 按钮打开规则管理面板。面板分为两个标签页：

7.2 互斥约束

互斥约束规定两种关系不能同时存在于同一有向节点对之间。创建新关系时系统自动检查。

  1. 打开规则面板，确保当前在 互斥约束 标签页。
  2. 在"关系 A"输入框中输入第一个关系名（如"管理"），在"关系 B"中输入第二个（如"控制"）。
  3. 点击 添加规则。规则列表显示当前所有互斥约束。
定义互斥：管理 ↔ 控制。之后如果 A→B 已有"管理"关系，再创建 A→B 的"控制"关系时系统会拦截并提示冲突。
互斥约束仅在创建新关系时检查，不会回溯检查已有关系。建议先定义规则再连线。只检查相同方向（同 source→target）的冲突。

7.3 顺承推理

顺承推理定义关系传递规则：当 A→B 存在关系 1，且 B→C 存在关系 2 时，系统自动创建 A→C 的推导关系。 推导出的关系是实际创建在画布上的，参与后续推理和贝叶斯分析。

  1. 切换到 顺承推理 标签页。
  2. 填写三个输入框：
    • 关系 1：第一段关系名（如"控制"）
    • 关系 2：第二段关系名（如"依赖"）
    • 推导结果：自动生成的关系名（如"支配"）
  3. 点击 添加规则。系统立即扫描已有连线进行推导。
画布现有：传感器 ──控制──▶ 控制器 ──依赖──▶ 执行器
规则：控制 + 依赖 → 支配
系统自动创建：传感器 ──支配──▶ 执行器。
删除推理规则不会删除已推导出的关系，只会阻止未来继续推导。如需清理，请手动删除。 推理规则要求关系名称逐字逐符完全匹配（包括大小写和标点）。
输入框支持自动补全（基于画布已有的关系名称）。建议先规划好关系名称字典，再设计规则，避免因名称不一致导致规则失效。

8. 贝叶斯因果分析

8.1 概述

贝叶斯因果分析是本系统的高级分析功能。它基于您构建的本体关系网络，使用Noisy-OR 概率组合模型， 对因果链条进行量化分析。分析功能位于"分析"工作台标签页。

分析面板包含三个标签页：向上追溯、向下传播、全局概览。

8.2 关系权重配置

在进行因果分析之前，需要为每种关系类型配置因果权重（0-1）。 权重表示当上游事件发生时，通过该关系传播到下游的概率。

  1. 在建模标签中，点击 关系权重配置 按钮。
  2. 在弹出面板中，为每种关系类型输入权重值（0 到 1 之间）。
  3. 点击保存。权重配置随项目保存。
在机械加工工艺本体中：控制 → 0.9（控制关系传播概率极高）、影响 → 0.5（影响关系中等概率）、 依赖 → 0.3（依赖关系较低概率）。
默认权重为 0.65。如果某种关系类型未配置权重，系统会在分析时使用默认值。每条关系连线也可以单独设置权重（创建时或选中后在详情面板中调整）。

8.3 向上追溯分析

向上追溯分析给定目标节点的所有上游影响因素。 系统沿因果链反向追踪（通过 DFS 深度优先搜索），找出所有可能影响该目标的上游节点， 并通过 Noisy-OR 模型计算每个因素对目标的综合影响概率。

  1. 在贝叶斯分析面板中，切换到 向上追溯 标签。
  2. 在下拉列表中选择一个目标节点。
  3. 点击 开始追溯。系统展示所有影响该目标的因素。

结果展示包括：因素名称、影响等级（高/中/低）、综合概率百分比、概率条可视化、每条因果路径及其单独概率。

目标节点：工件精度
追溯结果：
· 刀具磨损 → 高影响 → 78%（路径：刀具磨损 → 切削力变化 → 工件精度）
· 冷却液温度 → 中影响 → 45%（路径：冷却液温度 → 热变形 → 工件精度）

8.4 向下传播分析

向下传播分析给定源节点的所有下游影响范围。 系统沿因果链正向追踪（BFS 广度优先搜索），找出该因素可能影响的所有下游节点及概率。

  1. 切换到 向下传播 标签。
  2. 在下拉列表中选择一个因子节点（源）。
  3. 点击 开始传播。
源节点：冷却液浓度
传播结果：
· 切削温度 → 高影响 → 82%
· 刀具寿命 → 中影响 → 55%
· 工件表面粗糙度 → 中影响 → 40%

8.5 全局概览分析

全局概览扫描整个因果网络，自动识别：

  • 影响力排名：影响力最高的节点（出度最多、权重最大）。
  • 脆弱性排名：受影响最多的节点（入度最多、暴露最广）。
  • 最强因果链：网络中概率最高的几条因果路径。
全局概览分析完成后，系统会自动在画布上高亮显示最强因果路径中的节点，帮助您快速聚焦关键因果链条。 关闭分析面板后高亮自动清除。

8.6 Noisy-OR 概率模型简介

本系统使用Noisy-OR模型来组合多个独立原因对同一结果的综合影响。主要思想：

  • 每条因果边有一个独立的影响概率（权重）。
  • 多个上游因素对同一目标的影响被视为独立因果机制。
  • 综合概率 = 1 - (1-p₁) × (1-p₂) × ... × (1-pₙ)，pᵢ 为每条路径的概率。
  • 影响等级划分：≥70% 为高影响、40%~70% 为中影响、15%~40% 为低影响、<15% 为极低。
如果两个因素分别以 60% 和 50% 的概率影响同一目标：
综合概率 = 1 - (1-0.6) × (1-0.5) = 1 - 0.4 × 0.5 = 0.8 = 80%（高影响）。

9. 多视角浏览

9.1 概述

主内容区顶部有三个视角标签：概览（网格图标）、数据映射（数据库图标）、 本体推理（网络图标）。每个视角提供不同的信息查看方式，适合不同角色的用户。

9.2 概览视角（管理者仪表盘）

概览视角面向管理者和决策者，提供项目的全局摘要视图：

  • 摘要卡片：显示本体概念数、推理关系数、数据底座数三个核心指标。
  • 数据源分组：按数据源类型（达梦数据库 / Excel / CSV / 人工维护）分组展示数据来源卡片。
  • 数据-本体关联图：SVG 二分图，展示数据源与本体概念之间的对应关系，一目了然地看到哪些概念来自哪个数据源。
概览视角是只读的，不能在此编辑本体或关系。如需编辑，请切换到"本体推理"视角。

9.3 数据映射视角（数据血缘脑图）

数据映射视角以树形脑图的方式展示从业务域到数据字段的完整映射链：

  • 五级层次：根节点 → 业务域 → 本体概念 → 数据源 → 字段映射。
  • 交互式 SVG：支持拖拽平移、滚轮缩放、节点折叠/展开。
  • 高亮联动：鼠标悬停在某个节点上时，其所有祖先节点同时高亮，帮助追踪数据血缘。
  • 图例栏：底部彩色图例标识各层级节点的含义。
  • 布局算法：使用手动树布局算法，节点间距自适应调整。
场景：查看"热源子系统"下所有概念的数据来源。
在脑图中展开"热源子系统"业务域，可以看到其下所有概念（锅炉、换热器等）， 以及每个概念对应的数据源和具体字段。

11. 常见问题

11.1 连线箭头被本体卡片遮挡

系统使用几何算法确保连线端点精确落在卡片边框上。以下情况可能导致视觉遮挡：

  • 节点间距过近（建议水平间距至少 150px）。
  • 多个节点重叠。请拖拽节点分开。

11.2 点击连线没有反应

请点击连线上方的关系名称文字标签（白色圆角背景），该区域有 20px 的隐形点击热区。或放大画布后再操作。

11.3 顺承推理规则不生效

请依次检查：

  1. 规则中的关系名与画布上实际关系名逐字逐符相同（包括大小写、空格、中英文标点）。
  2. 路径方向正确：关系 1 的 target 等于关系 2 的 source（中间节点必须相同）。
  3. 关系顺序正确："关系 1"在前，"关系 2"在后。
  4. 推导结果关系尚不存在（不重复创建）。
  5. 中间节点未被折叠隐藏。

11.4 互斥约束不生效

互斥约束仅在创建新关系时检查，不会回溯已有关系。建议先定义规则再连线。只检查同方向（同 source→target）的冲突。

11.5 贝叶斯分析结果为空或不合理

  1. 确认已为相关关系类型配置了权重（在"关系权重配置"中设置）。
  2. 确认目标/源节点在因果网络中存在连接路径（孤立节点无法分析）。
  3. 向上追溯和向下传播分别使用 DFS 和 BFS 算法，确保方向正确。

11.6 数据备份与恢复

直接复制 backend/projects/ 整个目录即可完成备份。恢复时将备份覆盖回去即可（操作前关闭后端服务）。

建议在重要操作前（批量删除、大规模连线、导入数据前）先备份 projects/ 目录。

11.7 画布加载不出来或显示空白

  1. 确认后端服务正在运行（默认 http://127.0.0.1:8000）。
  2. 检查侧边栏状态栏消息，确认没有网络错误。
  3. 尝试点击"刷新画布"按钮。
  4. 检查项目列表是否正常加载。
  5. 如果使用了平移+缩放，画布可能被移出视野。点击右下角 ⌂ 重置按钮。

11.8 数据库连接失败

  • 模拟达梦：确认 backend/projects/sim_dameng.db 文件存在且未损坏。
  • 真实达梦：确认主机地址、端口（默认 5236）、用户名、密码正确；确认网络可达；确认 dmPython 驱动已安装。

11.9 状态栏消息解读

消息特征含义需要留意？
包含"已"字操作成功否，正常提示
包含"失败"字操作出错是，请检查网络或后端状态
"就绪"系统待命否，正常初始状态
"连线模式：…"正在连线模式否，提示当前模式
"加载失败"网络或后端异常是，请检查后端是否运行
"同步失败"保存数据出错是，数据可能未保存

11.10 典型工作流场景

场景推荐流程
从零建模新建项目 → 创建核心本体 → 添加子本体构建层次 → 建立关系连线 → 设置逻辑规则 → 配置权重 → 贝叶斯分析
从数据库导入新建项目 → 数据导入标签 → 连接数据库 → 选择表/列 → 导入 → 切换到建模标签补充关系
从 Excel 导入上传 Excel/CSV → Schema 预览 → 勾选并导入 → 调整布局 → 补充关系连线
模板导入准备符合模板格式的 Excel → 数据导入标签 → 选择模板模式 → 导入实体关系+属性+规则
因果分析配置权重 → 分析标签 → 选择目标节点 → 向上追溯/向下传播/全局概览 → 查看高亮路径
多版本管理为每个版本创建独立项目 → 项目间切换对比 → 定期备份 projects 目录

11.11 环境部署说明

本系统由前端（Vue3 + Vite）和后端（Python FastAPI）两部分组成：

开发环境启动

# 后端
cd backend
pip install -r requirements.txt
python app/main.py
# 服务运行在 http://0.0.0.0:8000

# 前端
cd frontend
npm install
npm run dev
# 开发服务器运行在 http://localhost:5173

生产部署

前端构建后静态文件部署到 Nginx 或其他 Web 服务器：

cd frontend
npm run build
# 将 dist/ 目录部署到 Web 服务器

后端可直接运行 FastAPI，或通过 Docker 容器化部署（推荐）。

帮助文档访问

本帮助文档（help.html）部署在前端 public/ 目录下，开发时可通过以下 URL 访问：

http://localhost:5173/help.html

在系统主界面中，点击侧边栏底部的 帮助文档 按钮即可在新标签页中打开。

11.12 系统限制与注意事项

  • 本体和关系的数量理论上没有限制，但节点数极大（数千以上）时渲染性能会下降。
  • 系统不提供撤销/重做功能，删除操作请谨慎。建议定期备份 projects/ 目录。
  • 父子关系由系统自动管理，不可手动创建或删除。
  • 画布的平移和缩放状态不会持久化，刷新后恢复默认。
  • 仅管理者角色可编辑本体和导入数据，普通用户为只读模式。
