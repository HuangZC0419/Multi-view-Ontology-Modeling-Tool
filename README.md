# 多视角本体建模工具 (Multi-view Ontology Modeling Tool)

基于 OCR 识别与贝叶斯网络的多视角本体建模系统，支持从图片、文档、Excel、数据库中自动抽取本体实体，在可视化画布中进行本体建模，并通过贝叶斯因果网络进行影响分析。

## 功能特性

### 本体可视化建模
- 画布拖拽创建本体节点，支持父子层级关系（`is-a` 自动连线）
- 任意两节点间手动拉线建立自定义关系
- 节点/关系的增删改查，自动保存至后端

### 多源数据导入
- **OCR 图片识别**：上传图片（png/jpg/bmp/webp），自动识别文字并提取中文名词实体
- **文本导入**：上传 txt/md 文件，分词抽取名词作为候选本体
- **CSV 导入**：上传 CSV 文件，自动解析行列数据并提取实体
- **Excel 数据源**：上传 xlsx/xls 文件，解析表结构并映射为本体节点与属性
- **数据库接入**：支持达梦（DM）数据库连接，读取表/列结构生成本体

### 贝叶斯网络影响分析
- 关系权重配置：为每条关系边设置因果概率权重
- **向上追溯**：查找目标节点的所有上游影响因素，计算因果路径概率
- **向下传播**：评估源节点对所有下游节点的潜在影响
- **全局概览**：影响力排名、脆弱性排名、最强因果链分析
- 推理规则自动传播：基于传递性规则自动推导间接关系

### 多视角数据视图
- **领导视角**：数据源覆盖概览、业务域分布、来源统计
- **工程师视角**：本体与数据源的字段级映射详情
- **工艺视角**：完整本体图数据

### 多项目管理
- 支持创建/重命名/删除多个项目
- 每个项目独立存储本体图数据、推理规则、视角配置
- 内置机械加工工艺案例项目

### 用户认证
- 基于 Excel 的用户名/密码登录系统
- 支持角色区分（admin/user）

## 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | Vue 3 + Vite |
| 后端 | Python + FastAPI |
| OCR | RapidOCR (onnxruntime, 本地离线) |
| NLP | jieba 分词 |
| 图像 | OpenCV |
| 数据库 | 达梦 DM 连接器 |
| 认证 | openpyxl (Excel 用户表) |
| 打包 | PyInstaller (单文件 exe, Win7 兼容) |

## 快速启动

### 开发模式

**1. 启动后端**

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python app\main.py
```

后端启动后访问 `http://localhost:8000`，API 文档见 `http://localhost:8000/docs`。

**2. 启动前端**

另开一个终端：

```bash
cd frontend
npm install
npm run dev
```

浏览器访问 `http://127.0.0.1:5173`。

### 打包部署

使用 `start.bat` 一键构建前端并启动后端：

```bash
start.bat
```

或手动构建：

```bash
cd frontend && npm run build
cd ..
pyinstaller OCR-Benti.spec
```

生成的 `dist/OCR-Benti.exe` 为单文件可执行程序，包含前端、后端、OCR 模型等全部依赖，可直接在 Windows 7+ 上运行。

## 使用说明

### 本体建模
1. 登录系统后进入画布界面
2. 点击 **"新增本体"** 创建节点，拖拽调整位置
3. 选中节点后点击 **"新增子本体"** 创建父子层级关系
4. 选中节点后点击 **"关系连线"**，再点击目标节点并输入关系名建立自定义关系
5. 右键节点可编辑、删除或重新挂载父节点

### OCR 导入
1. 侧边栏上传图片/txt/csv 文件
2. 点击 **"开始识别"** 获取候选词条（支持表格/文本自动识别）
3. 勾选需要的词条后点击 **"一键导入"** 创建本体节点

### 数据源接入
1. 通过侧边栏 **"数据源"** 面板上传 Excel/CSV 文件或连接数据库
2. 预览表结构和数据
3. 勾选需要的表和列，一键生成本体节点与属性

### 贝叶斯分析
1. 在 **"贝叶斯网络"** 面板配置关系权重
2. 选择目标节点进行 **向上追溯**，查看影响因素及因果路径
3. 选择源节点进行 **向下传播**，评估影响范围
4. 查看 **全局概览** 获取网络整体分析结果

### 推理规则
- 在规则面板配置传递性推理规则（如：A→控制→B, B→影响→C ⇒ A→决定→C）
- 配置互斥规则防止冲突关系
- 新建关系时自动触发推理并生成间接关系边

### 多项目管理
- 顶部栏切换/创建/重命名/删除项目
- 每个项目独立保存本体图、推理规则、视角配置
