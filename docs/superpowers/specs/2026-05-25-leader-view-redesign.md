# 概览页面（LeaderView）重新设计

## 背景

当前 LeaderView 展示的内容（数据源表格、覆盖度分析、域关系图）对领导视角无意义。领导只需要宏观了解：本体规模、关系数量、数据底座接入情况。

## 设计

### 区块 1：核心指标卡（3 个）

- 本体概念总数
- 推理关系总数
- 数据底座总数

### 区块 2：数据底座卡片（按类型分组）

按达梦/Excel/CSV 分组，每张卡片显示：
- 类型标签（达梦/Excel/CSV）
- 该类型数据源数量
- 每个数据源的名称
- 表数量和支撑的本体数量

### 区块 3：数据底座与本体支撑关系图

SVG 绘制，左侧数据源节点 → 右侧本体列表：
- 按数据源类型着色区分
- 每个数据源下列出其支撑的本体名称
- 本体数量多时默认折叠，显示前 N 个（如 8 个），超出部分显示"+M 个"展开按钮

## 数据来源

后端 API `/api/perspective/{id}/leader` 已返回所需全部数据，无需改后端。

## 移除的内容

- "业务域数"指标卡
- "数据源分布"指标卡
- 完整数据源表格
- 数据源与业务域关系图
- 数据源覆盖度百分比分析
