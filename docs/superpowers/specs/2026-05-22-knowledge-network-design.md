# 三视角知识网络重构 设计文档

日期: 2026-05-22 | 状态: 已确认

## 一、设计目标

将现有三个视角（概览/数据映射/本体推理）从"信息展示型"改造为"知识网络型"，每种视角使用不同的图可视化形态，统一以节点-关系图为核心表达。

## 二、关键决策

| 决策 | 选择 |
|------|------|
| 架构路线 | B — 三类独立图布局，共享数据 |
| 概览图形态 | **分组节点图** — 虚线框包围同域节点，域间粗线，数据源在边缘 |
| 数据映射图形态 | **思维导图** — 根左叶右，上业务下技术的混合树 |
| 本体推理图形态 | **力导向图** — 自动物理排布，节点大小=权重 |
| 工艺视角编辑 | 浏览=力导向自动布局；点击"编辑"→节点可拖拽固定位置+连线手柄 |
| 权限模型 | 维护者（可编辑/导入数据源）；浏览者（三视角只读） |
| 数据映射树层级 | 上半：业务域→本体概念；下半：概念→数据源·表·字段 |

## 三、三种图形态详述

### 3.1 概览视角 — 分组节点图（LeaderView）

- 4个业务域各为虚线圆角矩形框，包含域内本体概念节点
- 域间有依赖关系的用粗实线连接
- 数据源节点（达梦/Excel/CSV）呈现在画布外围边缘
- 节点大小按概念数比例缩放
- 交互：缩放、平移、hover 节点查看详情

### 3.2 数据映射视角 — 思维导图（EngineerView）

- 根节点=项目名（左侧），向右逐级展开
- 上半层级：业务域 → 本体概念
- 下半层级：本体概念 → 数据源类型标签 → 表名 → 字段名·类型
- 达梦=紫色标签，Excel=绿色标签，CSV=橙色标签
- 交互：展开/折叠节点、缩放、平移

### 3.3 本体推理视角 — 力导向图（ProcessView）

- 所有本体节点通过 d3-force 物理引擎自动排布
- 连线代表因果关系/影响关系，带箭头和标签
- 节点大小按贝叶斯权重/影响力缩放
- 浏览模式：节点自动排布，hover 查看详情
- 编辑模式（维护者）：节点可拖拽固定，出现连线手柄，侧边面板编辑关系

## 四、技术方案

### 需要新增
- d3-force 力导向图渲染引擎
- 树布局算法（思维导图）
- 分组节点图自绘 SVG
- 浏览/编辑双模式切换
- 权限控制（维护者/浏览者）

### 可复用
- 现有 NodeCard / EdgeLine（编辑模式）
- 后端数据模型（BusinessDomain / DataMapping）
- 视角配置 JSON
- PerspectiveTabs 导航栏
- API 端点（/api/perspective/*）
