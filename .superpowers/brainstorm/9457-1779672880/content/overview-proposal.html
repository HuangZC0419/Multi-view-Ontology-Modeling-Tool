概览页面重新设计方案

从"数据源明细表"转变为"宏观仪表盘"——简洁、直观、一眼看懂

❌ 当前设计（过度技术化）

4个区块：

1. 5个统计卡片（含无意义的"数据源分布"）

2. 完整数据源表格（名称/类型/连接/表/域）

3. 业务域-数据源关系图

4. 数据源覆盖度百分比分析

问题：信息过载，技术细节太多，领导看不懂也不需要看

✓ 建议设计（宏观仪表盘）

3个区块：

1. 核心指标卡 — 本体总数、关系总数、数据源总数

2. 数据底座概览 — 按类型展示接入的数据源（达梦/Excel/CSV各几个），每个显示名称和表数量

3. 模型基本信息 — 项目名、最后更新时间、创建时间

简洁：一屏看完，无需滚动，领导10秒理解全局

需要确认的问题

当前后端 API /api/perspective/{id}/leader 已经返回了 summary（含 node_count, source_count 等）、data_sources 列表和 domains 列表。重新设计概览页面不需要改后端。

具体来说，你希望：

A

精简指标卡

只保留 3-4 个核心数字：本体总数、关系总数、数据源总数、业务域数

B

数据底座卡片化

按达梦/Excel/CSV 分组，每组一张卡片展示数据源名称和包含的表数量，替代当前的大表格

C

模型统计信息

展示项目创建时间、最后更新时间、数据映射覆盖率（有多少本体已完成数据映射）

D

数据底座简易关系图

一个简单的图示，展示每个数据源连接到哪些业务域/本体，直观但不冗余
