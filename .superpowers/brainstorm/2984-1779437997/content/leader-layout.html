概览视角 — 四种图形态

业务域层级 + 数据源支撑关系的视觉化方式

A

力导向图（Force-Directed Graph）

业务域节点和数据源节点混排，关系用连线。节点大小代表概念数多少，连线粗细代表覆盖度。自动布局，节点互相推开避免重叠。

像 Neo4j 浏览器那种风格，一眼看出谁是核心

B

径向树 / 旭日图（Sunburst / Radial Tree）

中心是项目名，第一圈4个业务域扇区，第二圈15个本体概念，扇区大小按概念数比例。数据源用外围弧形标签标注。

层级关系一目了然，适合展示"什么属于什么"

C

思维导图（Mind Map / Tree）

中心根节点向右分叉，一级分支=4个业务域，二级分支=15个本体概念。每个概念旁边用小标签标出数据来源。

中国人熟悉的脑图风格，适合宏观梳理

D

分组节点图（Clustered Node Graph）

类似你现在画布里工艺视角的图，但把同域节点用虚线框围起来，域之间用粗关系线连接。数据源节点出现在画布边缘。

最接近工艺视角，风格统一
