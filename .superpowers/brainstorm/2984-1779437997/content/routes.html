三种风格路线 — "知识网络"的不同理解

先对齐概念：当你说"三个视角都做成知识网络风格"，意味着什么？

A

路线A：统一图画布 + 不同滤镜

三个视角底层是 同一张节点-关系图。切换视角只是改变：

  • 概览 → 只显示业务域节点（4个大节点），隐藏细节节点
  • 数据映射 → 显示本体节点 + 数据源节点 + "读取自"连线
  • 本体推理 → 完整图（现有画布）

本质：一个画布，三个图层

B

路线B：三类图布局 + 共享数据

同一套数据，但每种视角用不同的图结构展示：

  • 概览 → 树状图/旭日图（业务域→子概念层级）
  • 数据映射 → 二分图（本体节点 ↔ 数据库表，左右两列连线）
  • 本体推理 → 贝叶斯网络图（现有画布）

本质：三种图，同一份数据

C

路线C：知识图谱风格 + 卡片辅助

主体仍然是图（节点-关系），但不强制全用连线。可以用：

  • 概览 → 节点分组卡片（域节点 + 内含概念缩略图），关系用粗线表示域间影响
  • 数据映射 → 节点展开式（点击本体节点展开其数据源子节点层）
  • 本体推理 → 完整图（现有画布）

本质：图的变体，灵活混合

快速对比

路线A 路线B 路线C
实现复杂度 低 高 中
视角差异度 低 高 中
知识网络感 强 强 强
