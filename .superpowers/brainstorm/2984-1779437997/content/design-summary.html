三视角知识网络 — 设计方案总览

基于你的选择：B路线独立渲染 + 三种图形态 + 维护者/浏览者双权限

概览视角

领导 / 决策者

分组节点图 (D)

  • 虚线框包围同域节点
  • 域间粗线连接表关联
  • 数据源节点在画布边缘
  • 节点大小=概念数

只读浏览 ✦ 可缩放平移

数据映射视角

软件工程师

思维导图 (C)

  • 根在左，向右分叉展开
  • 上半：业务域→本体概念
  • 下半：概念→数据源·表·字段
  • 不同数据源不同颜色标签

只读浏览 ✦ 可展开/折叠节点

本体推理视角

工艺人员 / 知识维护者

力导向图 (A)

  • 所有节点自动力排布
  • 连线=因果/影响关系
  • 点击"编辑"切编辑模式
  • 编辑模式可拖拽+连线

浏览者只读 ✦ 维护者可编辑

关键设计决策汇总

决策项 选择
三视角架构 各自独立渲染，不要求跨视角节点引用
概览图形态 分组节点图 — 虚线框围域，域间粗线，数据源在边缘
数据映射图形态 思维导图 — 根左叶右，上半业务下半技术，混合层级
本体推理图形态 力导向图 — 自动排布，编辑时切手动模式
工艺视角编辑 浏览=力导向自动布局；点击"编辑"切可拖拽+连线手柄
权限模型 维护者：可编辑、可导入数据源；浏览者：三视角只读
交互层级 概览：缩放平移 · 数据映射：展开折叠+缩放 · 推理：力导向+编辑

实现要点

需要新增的能力

  • 分组节点图渲染引擎（d3-force 或自绘 SVG）
  • 思维导图渲染引擎（树布局算法）
  • 力导向图物理引擎（d3-force）
  • 工艺视角浏览/编辑双模式切换
  • 浏览者/维护者权限控制

可以复用的

  • 现有 NodeCard 组件（编辑模式下）
  • 现有 EdgeLine 组件（编辑模式下）
  • 后端数据模型（BusinessDomain / DataMapping）
  • 视角配置 JSON 文件
  • Tab 切换栏（PerspectiveTabs）
