模块一：数据源接入（数据库 + 文件导入 + 本体提取）

半自动流程：连接/上传 → 预览结构 → 勾选确认 → 生成本体

1. 连接器抽象层设计

DataSourceConnector 抽象基类 — 统一接口
class DataSourceConnector(ABC): # === 连接管理 === async def connect(config: dict) -> bool # 建立连接 async def disconnect() # 断开连接 async def test_connection(config: dict) -> bool # 测试连通性 # === 元数据读取 === async def get_tables() -> list[TableInfo] # 获取所有表/视图列表 async def get_columns(table: str) -> list[ColumnInfo] # 获取指定表的列信息 # === 数据预览 === async def preview_data(table: str, limit=10) -> list[dict] # 预览前N行 # === 数据结构 === class TableInfo: # name, schema, row_count, comment(达梦注释) class ColumnInfo: # name, data_type, nullable, is_pk, comment(达梦注释)

2. 达梦数据库连接器（DamengConnector）

技术方案

  • 驱动：dmPython（达梦官方Python驱动，conda benti-py38安装）
  • 连接参数：host, port(5236), user, password, schema
  • 元数据查询：通过 ALL_TABLES / ALL_TAB_COLUMNS 系统视图
  • 注释读取：达梦支持表和字段的 COMMENT 注释，用作本体名称的提示

模拟测试数据集

  • 创建一个达梦模拟数据库（SQLite 本地模拟达梦表结构）
  • 测试表：工艺参数表、设备台账、质检记录、物料清单
  • 用于开发阶段验证连接、读取、导入全流程
  • 真实达梦数据库在部署时替换

3. Excel/CSV 文件解析器（FileParser）

Excel（.xlsx / .xls）

  • 库：openpyxl（.xlsx）+ xlrd（.xls）
  • Sheet → 表：每个 Sheet 视为一个"概念源"
  • 第1行 → 列名：读取第一行作为字段名
  • 类型推断：读取前20行数据，按值推断类型

CSV（.csv）

  • 库：Python内置 csv 模块
  • 编码检测：chardet 自动检测 GBK/UTF-8
  • 分隔符检测：自动识别逗号/分号/制表符
  • 文件→表：每个 CSV 文件视为一个"概念源"

4. 本体提取逻辑（ontology_extract.py）

半自动流程：数据结构 → 本体概念
步骤 操作 说明
1. 获取结构 连接器返回 TableInfo[] + ColumnInfo[] 表名/Sheet名→候选父节点，列名/字段名→候选子节点
2. 预览确认 前端 SchemaPreview.vue 展示树形结构，用户勾选 勾选=导入，不勾选=跳过；可编辑节点名
3. 概念提取 生成 OntologyNode 列表 表名→父节点(name)，列名→子节点(name) + 属性(类型/注释)
4. 导入画布 POST /api/datasource/import 节点追加到当前项目，布局自动排列（网格/列式）

5. API 路由设计（routers/datasource.py）

新增 API 端点
# === 数据库连接 === POST /api/datasource/db/test ← 测试数据库连接 POST /api/datasource/db/connect ← 建立连接，返回 session_id GET /api/datasource/db/tables ← 获取表列表（需 session_id） GET /api/datasource/db/tables/{table}/columns ← 获取列信息 GET /api/datasource/db/tables/{table}/preview ← 预览数据 POST /api/datasource/db/disconnect ← 断开连接 # === 文件上传 === POST /api/datasource/file/upload ← 上传Excel/CSV，返回解析后的结构 GET /api/datasource/file/{file_id}/preview ← 预览文件数据 # === 本体导入（两种数据源共用）=== POST /api/datasource/import ← 提交勾选的表/列，生成本体节点并导入画布

6. 达梦模拟数据方案

开发阶段使用 SQLite 模拟达梦表结构

由于开发环境可能没有达梦数据库，使用 SQLite + 模拟适配器 来模拟达梦的元数据结构：

-- 模拟表结构（SQLite），完全模拟达梦系统视图的返回格式 CREATE TABLE all_tables (owner, table_name, comments, ...); CREATE TABLE all_tab_columns (owner, table_name, column_name, data_type, comments, ...); -- 录入测试数据：机械加工相关业务表 INSERT INTO all_tables VALUES ('BENTI', 'CUTTING_PARAM', '切削参数表'), ('BENTI', 'MACHINE_INFO', '机床信息表'), ('BENTI', 'TOOL_LIBRARY', '刀具库表'), ('BENTI', 'QUALITY_CHECK', '质检记录表');

SimDamengConnector 实现与真实 DamengConnector 完全相同的接口，查询此 SQLite 模拟库，方便开发调试。
