模块二：多视角查询系统 + 重新设计的 machining-demo

顶部 Tab 切换三个视角，同一套数据不同呈现方式

1. 三种视角定义

🏢 领导视角

关注：宏观概览与资源支撑

  • 展示业务域级大概念（不显示细节节点）
  • 显示数据支撑清单：哪些数据库/表支撑了哪些业务
  • 显示关键指标：节点总数、关系总数、数据源覆盖度
  • 卡片式布局，非图布局

💻 软件工程师视角

关注：数据来源与字段映射

  • 展示本体节点和底层数据映射（表名→字段名）
  • 点击节点可以看到数据血缘：此概念从哪个数据库、哪个表、哪个字段读取
  • 表格式+图混合布局

🔧 工艺人员视角

关注：本体关系与因果推理

  • 就是现有的本体画布：节点+关系+推理规则
  • 贝叶斯分析面板（上游追溯/下游传播/全局概览）
  • 完整的图编辑功能

2. 重新设计的 machining-demo — 三层架构

三层概念架构：业务域 → 本体概念 → 数据映射
第1层：业务域（领导视角可见）
4个顶层域，聚合下属所有本体节点
业务域 包含的本体概念 支撑数据库
🔧 设备与刀具域 机床刚性、刀具材料、刀具磨损、冷却液流量 MACHINE_INFO, TOOL_LIBRARY
⚙️ 工艺参数域 切削速度、进给量、切削深度、切削温度 CUTTING_PARAM
📏 质量检测域 表面粗糙度、尺寸精度、工件质量 QUALITY_CHECK
👥 人员与成本域 操作人员技能、加工效率、生产成本 OPERATOR_RECORD
第2层：本体概念（工艺人员视角可见）
15个本体节点 + 25条关系 + 贝叶斯分析，即现有案例内容
第3层：数据映射（软件工程师视角可见）
每个本体概念到底层数据库表/字段的映射关系
本体概念 数据源 表名 字段
机床刚性 🏦 达梦DB MACHINE_INFO rigidity_level, machine_model
切削速度 🏦 达梦DB CUTTING_PARAM cutting_speed, spindle_rpm
刀具材料 🏦 达梦DB TOOL_LIBRARY material_type, coating, hardness
表面粗糙度 🏦 达梦DB QUALITY_CHECK surface_ra, measure_date
工件材料硬度 📄 Excel material_spec.xlsx hardness_hb, material_grade
操作人员技能 🏦 达梦DB OPERATOR_RECORD skill_level, cert_type, exp_years
生产成本 📄 CSV cost_report.csv material_cost, labor_hour, total_cost

3. 数据模型扩展

扩展现有模型以支持多视角
# === 新增到 models.py === class PerspectiveType(str, Enum): LEADER = "leader" # 领导视角 ENGINEER = "engineer" # 软件工程师视角 PROCESS = "process" # 工艺人员视角 class BusinessDomain(BaseModel): # 业务域（领导视角用） id: str name: str # 域名称，如"设备与刀具域" ontology_node_ids: list[str] # 包含的本体节点ID databases: list[str] # 支撑的数据库/文件列表 class DataMapping(BaseModel): # 数据映射（工程师视角用） ontology_node_id: str # 本体节点 source_type: str # "dameng" | "excel" | "csv" source_name: str # 数据库名/文件名 table_name: str # 表名/Sheet名 field_mappings: list[FieldMapping] # 字段→属性映射 class FieldMapping(BaseModel): field_name: str # 数据库字段名 attribute_key: str # 对应的本体属性名 data_type: str # 数据类型

4. 视角 API 设计（routers/perspective.py）

多视角查询端点
# === 视角数据获取 === GET /api/perspective/{project_id}/leader ← 返回业务域列表+数据库支撑清单+统计指标 GET /api/perspective/{project_id}/engineer ← 返回本体节点+数据映射表（表名/字段/类型） GET /api/perspective/{project_id}/process ← 返回完整图数据（即现有 GET /api/graph 逻辑） # === 数据映射管理 === PUT /api/perspective/{project_id}/mappings ← 批量更新数据映射关系 GET /api/perspective/{project_id}/domains ← 获取业务域配置 PUT /api/perspective/{project_id}/domains ← 更新业务域配置
