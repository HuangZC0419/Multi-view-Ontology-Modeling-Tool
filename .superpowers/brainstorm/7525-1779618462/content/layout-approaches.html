A. 渐进折叠 + 语义缩放

根据缩放级别自动切换显示粒度。缩小看概念列表（带字段计数徽标），放大看完整映射树。滚轮缩放自然驱动层级过渡，无需点击。

优点

  • 滚轮驱动，流畅直觉
  • 始终保持概览感
  • 改动适中，保留现有布局

缺点

  • 语义缩放阈值需要调试
  • 中等层级时信息量可能跳跃

B. 网格卡片 + 抽屉详情

概览用卡片网格平铺所有概念（紧凑），点击某个概念在侧边抽屉展示完整字段映射。适合定位具体概念。

优点

  • 概览极紧凑，所有概念一目了然
  • 卡片搜索/排序很直观

缺点

  • 丢弃了树形层级关系
  • 需要大量重写，偏离现有架构
  • 概念间关系不可见
方案 C：优化现有树布局 + 迷你地图
🌲 树形布局保持，新增特性：
✓ 概念默认折叠（仅显示 20-50 个概念行）
✓ 每个概念旁显示 N字段 徽标
✓ 点击展开单个概念，查看数据源+字段
✓ 拖拽速度修复（除以scale）
✓ 可选：点击概念时自动居中+放大
🗺️ 迷你地图
蓝色框 = 当前视图
💡 这是对现有代码改动最小的方案 — 保留 SVG 树布局，叠加折叠/徽标/拖拽修复/迷你地图

C. 优化现有树 + 迷你地图（推荐）

保留当前 SVG 树形结构，增加：默认折叠概念、字段计数徽标、迷你地图导航、拖拽速度修复。改动最小、风险最低。

优点

  • 改动最小（~100行），风险可控
  • 保留层级关系可视化
  • 迷你地图解决"我在哪"问题

缺点

  • 概览时信息密度改善有限
  • 仍需要点击交互来展开细节
